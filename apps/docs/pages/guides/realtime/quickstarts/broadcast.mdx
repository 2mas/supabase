Realtime Broadcast follows the [publish-subscribe pattern](https://en.wikipedia.org/wiki/Publish%E2%80%93subscribe_pattern) where a client publishes messages to a channel based on a unique topic. For example, a user could send a message to a channel with topic `room-1`.

Other clients can receive the message in real-time by subscribing to the channel with topic `room-1`. These clients can continue to receive messages as long as they continue to be online and subscribed to the same channel topic.

A common use-case is sharing a user's cursor position with other clients in an online game.

## Quick start

Let's explore how to implement Realtime Broadcast so you can integrate it into your use case.

  <StepHikeCompact.Step step={1}>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={2}>
  
    <StepHikeCompact.Details title="Create a channel">

      A channel's topic can be anything except for `'realtime'`.

    </StepHikeCompact.Details>

      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={3}>
  
    <StepHikeCompact.Details title="Listen for messages">

  <StepHikeCompact.Step step={4}>
  
    <StepHikeCompact.Details title="Create the second client">

      This client will be used to send a message.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={5}>
  
    <StepHikeCompact.Details title="Create another channel">

      This channel's topic must match `channelA`'s.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```js
        const channelB = clientB.channel('room-1')
      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={6}>
  
    <StepHikeCompact.Details title="Send message">

            channelB.send({

## Broadcast options

There are additional Broadcast functionality that you can enable when creating a channel.

### Self-send messages

You can have a client send a message and then receive it.

<StepHikeCompact>

  <StepHikeCompact.Step step={1}>

    <StepHikeCompact.Details title="Create a client">

      Go to your Supabase project's [API Settings](https://supabase.com/dashboard/project/_/settings/api) and grab the `URL` and `anon` public API key.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```js
        import { createClient } from '@supabase/supabase-js'

        const clientC = createClient('https://<project>.supabase.co', '<your-anon-key>')
      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={2}>

    <StepHikeCompact.Details title="Create a channel with self-send capability">

      This channel will enable the ability to roundtrip a message by sending and then receiving it.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```js
        const channelC = clientC.channel('room-2', {
          config: {
            broadcast: {
              self: true
            }
          }
        })
      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={3}>

    <StepHikeCompact.Details title="Listen for messages">

      Set up channel handler to listen for messages.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```js
        channelC
          .on('broadcast', { event: 'test-my-messages' }, (payload) => console.log(payload))
      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={3}>

    <StepHikeCompact.Details title="Send message">

      Subscribe to channel and send a message.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```js
        channelC.subscribe((status) => {
          if (status === 'SUBSCRIBED') {
            channelC.send({
              type: 'broadcast',
              event: 'test-my-messages',
              payload: { message: 'talking to myself' },
            })
          }
        })
      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={4}>

    <StepHikeCompact.Details title="Same client receives message">

      `clientC` receives the message that it sent.

    </StepHikeCompact.Details>

  </StepHikeCompact.Step>

</StepHikeCompact>

### Acknowledge messages

You can confirm that Realtime received your message.

<StepHikeCompact>

  <StepHikeCompact.Step step={1}>

    <StepHikeCompact.Details title="Create a client">

      Go to your Supabase project's [API Settings](https://supabase.com/dashboard/project/_/settings/api) and grab the `URL` and `anon` public API key.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```js
        import { createClient } from '@supabase/supabase-js'

        const clientC = createClient('https://<project>.supabase.co', '<your-anon-key>')
      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={2}>

    <StepHikeCompact.Details title="Create a channel with acknowledgement capability">

      This channel will enable Realtime to confirm that it received your channel's sent message.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```js
        const channelD = clientD.channel('room-3', {
          config: {
            broadcast: {
              ack: true
            }
          }
        })
      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

  <StepHikeCompact.Step step={3}>

    <StepHikeCompact.Details title="Send message with acknowledgement">

      Subscribe to channel, send a message, and get acknowledgement that Realtime received it.

      If Broadcast config `ack` is not set to `true` when creating channel, the promise returned by call to `channelD.send` will resolve immediately.

    </StepHikeCompact.Details>

    <StepHikeCompact.Code>

      ```js
        channelD.subscribe(async (status) => {
          if (status === 'SUBSCRIBED') {
            const resp = await channelD.send({
              type: 'broadcast',
              event: 'acknowledge',
              payload: {},
            })

            console.log('resp', resp)
          }
        })
      ```

    </StepHikeCompact.Code>

  </StepHikeCompact.Step>

</StepHikeCompact>

## Client-side rate limit

There is a default client-side rate limit that enables you to send 10 messages per second, or one message every 100 milliseconds. You can customize this when creating the client:

```js
  import { createClient } from '@supabase/supabase-js'

  const clientE = createClient(
    'https://<project>.supabase.co',
    '<your-anon-key>',
    {
      realtime: {
        params: {
          eventsPerSecond: 20
        }
      }
    }
  )
```

By setting `eventsPerSecond` to 20, you can send one message every 50 milliseconds on a per client basis.

Learn more by visiting the [Quotas](/docs/guides/realtime/quotas) section.
